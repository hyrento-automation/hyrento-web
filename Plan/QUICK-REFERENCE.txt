PROMPT RULES (follow these to save tokens)
==========================================================================

✅ DO:
- One page or component per message
- Paste exact error messages when debugging
- Say which file to edit (don't make Claude guess)
- Confirm "working, move to next" before continuing
- Use the templates in /prompt-templates/

❌ DON'T:
- "Build the whole app" in one message
- Describe your brand colors more than once (use the brief)
- Ask Claude to "improve" or "refactor" without specifying what
- Start a new topic without confirming the previous one works

==========================================================================
CLAUDE CODE COMMANDS (terminal)
==========================================================================

Start a new project:
  npx create-next-app@latest my-app --typescript --tailwind --app

Install shadcn/ui:
  npx shadcn-ui@latest init

Add a shadcn component:
  npx shadcn-ui@latest add button
  npx shadcn-ui@latest add card
  npx shadcn-ui@latest add table
  npx shadcn-ui@latest add dialog
  npx shadcn-ui@latest add form

Install Supabase:
  npm install @supabase/supabase-js @supabase/ssr

Run dev server:
  npm run dev

Build and check for errors:
  npm run build

WHEN THINGS GO WRONG
==========================================================================

Component looks wrong visually:
→ Attach frontend-design SKILL.md and say "redesign this component 
  following the skill rules. Here is what's wrong: [describe]"

File read error:
→ Attach file-reading SKILL.md and say "retry reading this file"

TypeScript errors on build:
→ Use bug-fix-prompt.txt, paste the exact tsc error output

Supabase not connecting:
→ Check .env.local has both NEXT_PUBLIC_SUPABASE_URL and 
  NEXT_PUBLIC_SUPABASE_ANON_KEY set correctly
